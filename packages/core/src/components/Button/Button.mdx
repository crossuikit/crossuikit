# Button

A button component that follows Material-UI design principles and provides various customization options.

## Features

- Multiple variants (contained, outlined, text)
- Different sizes (small, medium, large)
- Support for icons
- Loading state
- Disabled state
- Full width option
- Custom colors
- Click handlers

## Usage

### Basic Usage

<Source
  language="tsx"
  code={`
import { Button } from '@crossuikit/core';

function MyComponent() {
  return (
    <Button
      variant="contained"
      onClick={() => console.log('Button clicked')}
    >
      Click Me
    </Button>
  );
}
  `}
/>

### With Loading State

<Source
  language="tsx"
  code={`
import { Button } from '@crossuikit/core';
import { useState } from 'react';

function LoadingButton() {
  const [loading, setLoading] = useState(false);

  const handleClick = () => {
    setLoading(true);
    setTimeout(() => setLoading(false), 2000);
  };

  return (
    <Button
      loading={loading}
      onClick={handleClick}
    >
      {loading ? 'Loading...' : 'Click to Load'}
    </Button>
  );
}
  `}
/>

## Best Practices

1. **Button Text**: Use clear, action-oriented text that describes what will happen when clicked.
2. **Button Placement**: Place primary actions on the right and secondary actions on the left.
3. **Button Size**: Use appropriate sizes based on the context and importance of the action.
4. **Loading State**: Show loading state for actions that take time to complete.
5. **Disabled State**: Disable buttons when the action is not available.
6. **Accessibility**: Ensure buttons have proper ARIA labels and keyboard navigation.
7. **Consistency**: Maintain consistent button styles throughout your application.
